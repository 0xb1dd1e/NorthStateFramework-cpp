Detailed Description

Represents a thread that has an event queue and dispatches events to their destinations.

Parameters
nsfEventThe event to queue.
isPriorityEventFlag indicating if the event should be queued to the back of the queue (false) or the front of the queue (true).
logEventQueuedFlag indicating if an event queued trace should be added to the trace log.

Queueing a priority event to a state machine will invalidate its UML run-to-completion semantics. Use this feature for custom event handlers only.

void terminate ( bool  waitForTerminated)
virtual

Terminates the thread by causing the execution method to return.

Parameters
waitForTerminatedFlag indicating if the method should wait until the thread is terminated (true), or if it should return immediately (false).

This method does not have to be called before deleting a thread, because the proper termination sequence is guaranteed by the destructor. However, this method is useful to guarantee that the thread is no longer active. If the waitForTerminated flag is set true, this method must not be called from its thread of execution.
