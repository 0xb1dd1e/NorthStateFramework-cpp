NSFEvent * copy ( bool  deleteAfterHandling)
virtual

Creates a deep copy.

Parameters
deleteAfterHandlingFlag indicating if the event should be deleted after handled by an event handler.
Returns
A copy of the event.

A common design pattern is to queue data event copies, each with its own unique data payload, to state machines for handling. The deleteAfterHandling flag is useful in this situation, because it allows new events to be created and queued to a state machine, letting the state machine delete the event once its handled.

Reimplemented in NSFDataEvent< DataType >, NSFDataEvent< NorthStateFramework::NSFXMLElement * >, and NSFDataEvent< NSFString >.

Gets the event id.

Transitions use the event id to match the trigger event. In other words, mulitple event objects with the same id can trigger a transition. Use the copy() method to create a new event object with the same id as an existing object.
